Section 01

What is R?

R is a free, open-source programming language built for statistical computing and data visualisation. It is widely used across government, research, and industry for data analysis.

  • Create charts, graphs, and visualisations from data
  • Perform statistical analyses and modelling
  • Read, clean, and manipulate datasets
  • Build interactive web applications (Shiny)
  • Generate reproducible reports combining code, results, and narrative (RMarkdown)

Key Concept: R vs RStudio

R is the engine — it performs the computation. RStudio is the interface — it provides a convenient way to interact with R. You need both installed, but you will only open RStudio (it starts R automatically). Think of R as the engine and RStudio as the dashboard.

Section 03

The RStudio Interface

When you open RStudio, you will see four panes. Each serves a specific purpose:

Source (Top-Left)

Your code editor. Write and edit R scripts here. It provides syntax highlighting and auto-completion.

  • ✏️ Write and edit .R script files
  • ✏️ Create RMarkdown .Rmd documents
  • ✏️ Build Shiny applications
Environment (Top-Right)

Your workspace inventory. Every variable, dataset, and function you create appears here.

  • 📊 View all variables and their values
  • 📊 Click datasets to view them as spreadsheets
  • 📊 Import data using the Import button
Console (Bottom-Left)

Where R runs. Output and errors appear here. You can also type commands directly.

  • ▶️ View output and error messages
  • ▶️ Type commands at the > prompt
  • ▶️ Install packages here
Files/Plots/Help (Bottom-Right)

The multi-purpose pane. Browse files, view plots, check packages, and read documentation.

  • 📁 Files — browse your file system
  • 📈 Plots — view generated charts
  • 📖 Help — read function documentation
  • 📦 Packages — view installed packages

RStudio's four-pane layout — Source, Environment, Console, and Files/Plots/Help

Section 04

R Basics & Syntax

Assignment (Creating Variables)

Use <- to store values. Think of it as "put this value into this name."

# Store a number x <- 42 # Store text (a "string") name <- "Hello World" # Store a list (a "vector") scores <- c(85, 90, 78, 92) # Print the values x # Returns: 42 name # Returns: "Hello World" scores # Returns: 85 90 78 92

Data Types

R has several basic types of data. The most common ones are:

# Numeric (decimal numbers) height <- 5.9 # Integer (whole numbers) age <- 25L # Character (text/strings) greeting <- "Hello" # Logical (TRUE / FALSE) is_active <- TRUE # Check the type class(height) # "numeric" class(greeting) # "character"

Vectors

A vector is an ordered list of items of the same type. The most basic data structure in R.

# Create with c() fruits <- c("apple", "banana", "cherry") temps <- c(72, 68, 75, 80, 69) # Access elements (R starts counting at 1) fruits[1] # "apple" fruits[2:3] # "banana" "cherry" # Quick statistics mean(temps) # Average: 72.8 sum(temps) # Total: 364 length(temps) # Count: 5

Data Frames

A data frame is like a spreadsheet — rows and columns. Most data in R is stored this way.

# Create a data frame records <- data.frame( name = c("Alice", "Bob", "Carol"), age = c(22, 25, 21), grade = c("A", "B", "A") ) # View as spreadsheet View(records) # Access a column records$name # Alice, Bob, Carol records[, 2] # 22, 25, 21 (2nd column) records[1, ] # First row (Alice's data)
Section 05

Running R Scripts

An R script is a text file (with a .R extension) containing R code. Here are the ways to run one:

A

Run Line by Line Recommended

Place your cursor on the line you want to run, then press Ctrl + Enter (Windows/Linux) or Cmd + Enter (Mac). The line executes in the Console.

B

Run the Entire Script

Press Ctrl + Shift + Enter or click the "Source" button at the top of the Source pane.

# Or use the source() command in the Console: source("my_script.R") # With the full file path: source("C:/Users/Name/Documents/my_script.R")
C

Run Selected Code

Highlight the specific lines you want to run, then press Ctrl + Enter. Only the selected code will execute.

D

Run from Command Line

# In Terminal / Command Prompt: Rscript my_script.R # Save output to a file: Rscript my_script.R > output.txt

How to Run a Colleague's Script

1

Open RStudio (double-click the RStudio icon)

2

Go to File → Open File… and navigate to the .R script

3

If the script requires packages, install them first (see Section 07)

4

Ensure any required data files are in the same folder as the script

5

Set the working directory: Session → Set Working Directory → To Source File Location

6

Press Ctrl + Shift + Enter to run the entire script

7

Check the Console for output and the Plots tab for charts

Section 06

Key Functions

Functions are reusable recipes — you provide inputs (arguments), and they produce a result. These are the functions you will encounter most frequently.

Importing & Exporting Data

read.csv()

Read a CSV file into R as a data frame.

data <- read.csv("my_data.csv")
read_excel()

Read an Excel file. Requires the readxl package.

library(readxl) data <- read_excel("my_data.xlsx")
write.csv()

Save a data frame to a CSV file.

write.csv(data, "output.csv", row.names = FALSE)
readLines()

Read text files line by line.

text <- readLines("document.txt")

Exploring Data

head()

Show first 6 rows of a dataset

tail()

Show last 6 rows

str()

Structure: type, rows, columns

summary()

Min, max, median, mean, quartiles

dim()

Dimensions (rows × columns)

names()

Column names

View()

Open spreadsheet viewer

nrow() / ncol()

Number of rows / columns

class()

Object type (data.frame, etc.)

Statistics

mean()

Average. Use na.rm=TRUE to ignore NAs

median()

Middle value when sorted

sd()

Standard deviation

var()

Variance

cor()

Correlation (-1 to +1)

table()

Frequency count of each value

lm()

Linear regression model

t.test()

Compare means between two groups

sum()

Add up all values

Visualisation

plot()

Base R scatter plot.

plot(x = data$age, y = data$score, main = "Age vs Score", xlab = "Age", ylab = "Score")
hist()

Histogram showing distribution.

hist(data$age, main = "Age Distribution", col = "steelblue")
ggplot() + geom_*()

Powerful plotting from the ggplot2 package.

library(ggplot2) ggplot(data, aes(x = age, y = score)) + geom_point() + geom_smooth(method = "lm") + labs(title = "Age vs Score")
barplot() / boxplot()

Bar charts and box plots.

boxplot(score ~ group, data = data, col = c("#0B5394", "#2C3E6B"))

Data Manipulation (dplyr)

library(dplyr) # The five main verbs of dplyr: data %>% # Start with your data filter(age > 20) %>% # Keep rows matching condition select(name, age, score) %>% # Keep specific columns mutate(category = ifelse(score > 80, "High", "Low")) %>% # Add new column arrange(desc(score)) %>% # Sort (highest first) summarise(avg = mean(score)) # Calculate summary

What is %>% ?

The %>% symbol is the pipe operator. It passes the result of the left side as the first argument to the right side. Read it as "and then". Example: data %>% filter() %>% select() means "Take the data, AND THEN filter it, AND THEN select columns."

Section 07

Installing Packages

Packages are add-ons that extend R's capabilities. Think of R as a phone and packages as apps — you install them once, then load them whenever needed.

Installing a Package

You only need to install a package once per computer:

# Install a single package install.packages("ggplot2") # Install multiple at once install.packages(c("dplyr", "tidyr", "readxl"))

Loading a Package

You must load a package every time you restart R:

# Load the package library(ggplot2) # Now use its functions ggplot(data, aes(x, y)) + geom_point()

Common Packages

Package Purpose Key Functions
ggplot2 Data visualisation ggplot(), geom_point(), geom_bar()
dplyr Data manipulation filter(), select(), mutate(), summarise()
tidyr Reshaping data pivot_longer(), pivot_wider()

Section 08

RMarkdown Documents

RMarkdown combines narrative text, R code, and results (tables, plots) in a single document. When you "knit" it, R runs all the code and produces a polished report in HTML, PDF, or Word format.

Creating an RMarkdown Document

1

In RStudio: File → New File → R Markdown…

2

Enter a title, select output format (HTML, PDF, or Word), and click OK

3

Write text in Markdown and R code in code chunks

4

Click "Knit" or press Ctrl + Shift + K

Code Chunk Shortcut

Press Ctrl + Alt + I (Windows) or Cmd + Option + I (Mac) to insert a code chunk.

RMarkdown Structure

--- title: "Analysis Report" author: "Your Name" output: html_document --- ## Introduction This is regular text. It will appear in the final report as formatted prose. ```{r} # This is a code chunk — R will run this library(ggplot2) data <- read.csv("data.csv") summary(data) ``` ## Results The chart below shows the findings: ```{r} ggplot(data, aes(x = var1, y = var2)) + geom_point() ```

Rendering an RMarkdown File

In RStudio

  1. Open the .Rmd file
  2. Ensure required packages are installed
  3. Click "Knit" at the top of the Source pane
  4. A preview will appear; the output file is saved alongside the .Rmd

From Console

library(rmarkdown) render("report.Rmd") # Specify output format render("report.Rmd", output_format = "pdf_document")
Section 09

Shiny Apps

Shiny is an R package for building interactive web applications from R. Users interact with sliders, dropdowns, and buttons, and see results update in real time — no web development knowledge required.

Shiny App Structure

Every Shiny app has two components:

  • UI User Interface — defines the layout, input controls, and output displays
  • Server Server Logic — contains the R code that produces outputs based on user inputs

Creating a New Shiny App

In RStudio: File → New File → Shiny Web App… → Enter a name → Create

library(shiny) # ── UI: What the user sees ── ui <- fluidPage( titlePanel("Sample Dashboard"), sidebarLayout( sidebarPanel( sliderInput("bins", "Number of bins:", min = 5, max = 50, value = 20) ), mainPanel( plotOutput("distPlot") ) ) ) # ── Server: What R computes ── server <- function(input, output) { output$distPlot <- renderPlot({ x <- faithful[, 2] bins <- seq(min(x), max(x), length.out = input$bins + 1) hist(x, breaks = bins, col = "#0B5394", border = "white") }) } # ── Run the app ── shinyApp(ui = ui, server = server)

How to Run a Shiny App

From RStudio

  1. Open the app.R file
  2. Ensure shiny is installed
  3. Click "Run App" at the top of the Source pane
  4. A browser window opens with the app
  5. To stop: press Esc or click the Stop sign in the Console

From Console

library(shiny) runApp("path/to/app_folder") # Open directly in browser runApp("app_folder", launch.browser = TRUE)
Section 10

Troubleshooting

Common errors and their solutions:

Error: object 'x' not found

Meaning: The variable doesn't exist. You may have forgotten to run the line that creates it, or misspelt the name (R is case-sensitive).

Fix: Run the script from the beginning, or check spelling.

there is no package called 'ggplot2'

Meaning: The package has not been installed on this computer.

Fix: Run install.packages("ggplot2") first.

could not find function "ggplot"

Meaning: You're calling a function from a package that is not loaded.

Fix: Add library(ggplot2) at the top of your script.

file 'data.csv' does not exist

Meaning: R cannot find the file. The working directory is probably incorrect.

Fix: Use Session → Set Working Directory → To Source File Location or provide the full file path.

Warning: Removed N rows containing missing values

Meaning: A warning, not an error. The code ran, but some data had missing values (NA) that were skipped.

Fix: Usually safe to ignore. Check with sum(is.na(data)) if unexpected.

unexpected symbol in "my data"

Meaning: Syntax error — a missing comma, quote, parenthesis, or a space in a variable name.

Fix: Use underscores: my_data not my data. Check for missing punctuation.

General Tips

  • ✦ Always read the full error message
  • ✦ Run scripts from the top — R executes in order
  • ✦ Check your working directory: setwd()
  • ✦ Use ?function_name for help
  • ✦ Restart R: Session → Restart R
  • ✦ Search the exact error message online

Section 12

Glossary

Vector

An ordered list of items of the same type. The most basic data structure in R.

Data Frame

A table with rows and columns (like a spreadsheet). The standard structure for analysis.

Function

Reusable code that takes inputs (arguments) and produces an output. Called with parentheses: name()

Package / Library

A collection of functions extending R. Install once with install.packages(), load with library().

Working Directory

The folder where R looks for files. Check with getwd(), change with setwd().

Console

The interactive command line where code is executed and results displayed.

Script

A text file (.R) containing R code that can be saved and run as a unit.

Knit

The process of converting an RMarkdown file into a finished document by running all code chunks.

Pipe (%>%)

Passes the left side's result as the first argument to the right side. Read as "and then."

NA

"Not Available" — R's representation of missing data. Use na.rm = TRUE to ignore in calculations.

Tibble

A modern data frame from the tidyverse. Prints more neatly and is stricter about types.

CRAN

Comprehensive R Archive Network. The official repository for R and its packages.
